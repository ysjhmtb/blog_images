 1)

 2)

    template<typename T>
    void MyArray<T>::print() {
        for (int i = 0; i < m_length; i++)
            std::cout << m_data[i] << " ";
        std::cout << std::endl;
    }

    를 MyArray.cpp에 위치시키고 빌드하면,

    Undefined symbols for architecture x86_64:
      "MyArray<char>::print()", referenced from:
          _main in main.cpp.o
    ld: symbol(s) not found for architecture x86_64
    clang: error: linker command failed with exit code 1 (use -v to see invocation)

    와 같은 에러가 발생한다.

    왜냐하면 MyArray.cpp에서 void MyArray<T>::print()를 인스턴스화 할때, T를 char로 처리해야 할지,
    int로 처리해야 할지를 알 수가 없기 때문이다.

    3)
    main.cpp에 #include "MyArray.cpp"를 하면 일단 빌드 에러는 면한다. 하지만 추후 코드 규모가 커지면
    걷잡을 수 없는 문제가 발생한다. 채택해서는 안될 해결책이다.

    4)
    해결책은 explicit instantiation 이다.
    MyArray.cpp 에 다음과 같이 입력하면 빌드 에러를 면할 수 있다.

    // explicit instantiation
    template void MyArray<char>::print();
    template void MyArray<double>::print();

    하지만 처리해야 할 함수가 많다면 번거롭다.

    5) class 자체를 explicit instantiation 하면 된다.

    MyArray.cpp에

    // explicit instantiation
    template class MyArray<char>;
    template class MyArray<double>;

```C++

// MyArray.h

#ifndef DEC_MYARRAY_H
#define DEC_MYARRAY_H

#include <assert.h>
#include <iostream>


template<typename T>


class MyArray {
private:
    T m_length;
    T *m_data;

public:
    MyArray() {
        m_length = 0;
        m_data = nullptr;
    }

    MyArray(int length) {
        m_data = new T[length];
        m_length = length;
    }

    ~MyArray() {
        reset();
    }

    void reset() {
        delete[] m_data;
        m_data = nullptr;
        m_length = 0;
    }

    T &operator[](int index) {
        assert(index >= 0 && index < m_length);
        return m_data[index];
    }

    int getLength() {
        return m_length;
    }

    void print();
};





#endif //DEC_MYARRAY_H

```
